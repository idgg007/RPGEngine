### 架构设计

这是一个2D的基于$raylib$的游戏引擎。

支持如下函数：

```c
void *CreatWorld(); // 创建一个世界（场景）
void *CreatObject(); // 创建一个物体
void AddEnvironment(void *world,void *object);//
```



##### object类

提供多个图片的附加（`Draw`函数附带图片序号），在$Draw$的时候自动根据视野范围与世界坐标轴，计算出图片的显示范围，对于场景的切换，支持自带时间倍率的mov（单次移动）和加速度叠加和判断碰撞和力反馈（单向）。碰撞箱未考虑也不计划考虑力矩旋转的因素，但是碰撞箱是用线段表示的。产生的加速度仅取决于撞击深度（碰撞线条交错的深度）和object的position与线条的相对位置（不考虑重合的情况）。不过应当提供两个种类（弹性与非弹性），非弹性碰撞的时候应当同时消除物体在垂直于碰撞线段方向上的速度，弹性碰撞仅提供回弹力。（或许也应当支持摩擦力的计算）

##### world类

其中包含多个object类，这个类将object分为两类，但是两类的Object用同一个`vector<object*>` 存储，在渲染的时候从 `size() - 1` 到 `0` 序号依次显示，同时具有一个 `vector<bool> type` 表示这个object是环境物体还是可以动的物体，对于所有的可移动的物体，会收到其它所有物体的力，对于环境物体则不会受到任何力，在模拟开始时，会给所有可移动物体带上重力加速度，然后对每一个可移动物体判断碰撞，然后对每一个可移动物体去mov。

#### 运行逻辑

在 Init的时候启动渲染线程和模拟线程（10次每秒刷新计时）（会检测并等待是否有Show状态下的world），当世界Show状态被改变时，会检查当前世界的类型，同时更改一些信息。

```c
void *display; // 表示正在显示的世界的指针
void *simulate_display; // 表示跟随显示模拟的指针
void *simulate_run; // 表示每秒模拟100次的世界指针
```

上面的那些变量保证线程安全，保证同一时间，模拟和渲染最多只会有一个运行。保证在世界失去Show的状态时也会停止模拟
